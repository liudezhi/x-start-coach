# 通用概念学习与教程模板

这些模板不绑定具体模型、工具名或平台。按当前环境可用能力执行；缺少浏览、文件或绘图能力时，使用 `SKILL.md` 中的降级策略。

## 学习一个概念

```text
你是知识讲解专家和教学型作者。我要学习「{概念}」，目前是{学习水平}，最终希望能把它准确地讲给「{目标读者}」。

请根据我的目标选择合适深度，并覆盖：
1. 一句话定义
2. 它为什么出现、解决什么问题
3. 一个生活类比
4. 这个类比在哪里会失真
5. 它在更大知识体系中的位置
6. 它如何运作
7. 一个具体例子
8. 与 2-3 个相邻概念的区别
9. 常见误区、边界和失效条件
10. 5 个由浅入深的自测问题
11. 一段准确、好记的总结

先讲直觉，再讲术语。区分事实、推断和有争议的观点。
如果内容依赖最新信息或属于高风险领域，请先核实权威来源。
输出方式：{直接回答 / Markdown 教程 / 保存为文件}
```

## 对比容易混淆的概念

```text
请对比「{概念 A}」「{概念 B}」和「{概念 C，可选}」。

要求：
1. 先给一句话结论
2. 用相同维度比较定义、目的、体系位置、输入输出、机制、适用场景、例子和边界
3. 如果当前环境具备图片生成能力，生成一张最合适的概念关系图；否则给出 Mermaid 图
4. 提供正例、反例和容易选错的场景
5. 解释最常见的混淆原因
6. 给出可操作的选择标准
7. 用准确但好记的方式总结

不要把某个概念的目的与另一个概念的实现方式放在同一维度比较。
输出方式：{直接回答 / Markdown 教程 / 保存为文件}
```

## 苏格拉底式学习

```text
你是我的概念学习教练。围绕「{概念}」连续问我 6 个问题，从复述、区分和举例，逐步进阶到机制、应用和反例。

每次只问一个问题。等我回答后：
1. 判断准确程度
2. 保留正确部分
3. 修正最重要的错误或遗漏
4. 给出更适合教学的表达
5. 再问下一题

在我回答前，不要直接公布完整答案。
```

## 生成教程

```text
请把「{概念或主题}」整理成一篇面向「{目标读者}」的精品教程。

要求：
1. 从真实困惑切入
2. 先给结论，再逐层深入
3. 包含定义、背景、类比、类比边界、机制、例子、对比和误区
4. 使用最合适的图、表或 Mermaid 图表达关系
5. 区分事实、推断和有争议的观点
6. 语言通俗，但不牺牲准确性
7. 结尾包含自测题和记忆总结
8. 对最新或高风险内容核实权威来源
9. 使用 Markdown 输出
10. 如果当前环境具备图片生成能力，默认生成一张教学图片，直观展示概念、关系、机制或架构；否则使用 Mermaid 图

交付方式：{直接返回 Markdown / 保存为工作区文件}
视觉要求：默认生成一张图片；只有我明确说“不需要图片”时才跳过
```

## 审查教程

```text
请作为知识教程审稿人检查下面的内容。

检查：
1. 定义是否准确
2. 是否缺少必要背景、条件或适用范围
3. 类比是否会误导，是否说明了类比边界
4. 是否混淆类别、角色、过程、标准、产品、方法或实例
5. 是否适合目标读者
6. 是否存在过时信息或需要权威来源支持的结论
7. 是否真正由浅入深
8. 图表关系是否与正文一致

输出：
问题清单 -> 问题影响 -> 修改建议 -> 可直接替换的改写 -> 仍需核实的内容

目标读者：{目标读者}
内容：
{教程内容}
```

## 生成知识卡片

```text
请把「{概念}」压缩成一张面向「{目标读者}」的知识卡片。

包含：
- 一句话定义
- 解决的问题
- 核心机制
- 一个例子
- 与最容易混淆概念的区别
- 一个常见误区
- 一个判断或应用口诀

限制：{字数或版面限制}
```
